# WireGuard Configurations

## Описание режимов

### Forced Tunnel (`wg-forced-tunnel.conf.example`)
Весь трафик клиента (включая доступ в Интернет) заворачивается в туннель.
- **Плюсы:** Полный контроль над трафиком, защита в публичных Wi-Fi.
- **Минусы:** Нагрузка на VPN-сервер, замедление доступа к "тяжелым" ресурсам (видео).

### Split Tunnel (`wg-split-tunnel.conf.example`)
В туннель идут только запросы к корпоративным ресурсам (Intranet, RDP, File Shares).
- **Плюсы:** Снижение нагрузки на сервер, высокая скорость личного трафика.
- **Минусы:** Личный трафик не защищен корпоративным периметром.

## Журналирование
Для соответствия требованиям безопасности (в т.ч. 152-ФЗ) необходимо настроить логирование на сервере:
- Факт подключения/отключения (handshake).
- IP-адрес источника.
- Объем переданных данных.

*Примечание:* WireGuard по умолчанию "тихий" и хранит мало логов. Рекомендуется использовать wrapper-скрипты или динамический дебаг (`echo module wireguard +p > /sys/kernel/debug/dynamic_debug/control`) для сбора событий.
