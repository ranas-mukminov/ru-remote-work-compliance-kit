# Contributing to ru-remote-work-compliance-kit

Мы рады вашим пулл-реквестам!

## Основные правила

1. **TDD (Test Driven Development):**
   - Любой новый функционал должен начинаться с написания тестов.
   - Сначала тесты -> потом код.
   - Перед отправкой PR убедитесь, что все тесты проходят.

2. **Безопасность:**
   - Не коммитьте реальные ключи, пароли или персональные данные.
   - Используйте примеры (`.example`) для конфигов.

3. **Проверка кода:**
   - Запустите `scripts-dev/dev_run_all_tests.sh` перед созданием PR.
   - Этот скрипт запустит линтеры и тесты.

## Структура PR

- Опишите, что делает ваш PR.
- Укажите, какие issue он закрывает.
- Добавьте информацию о том, как вы тестировали изменения.
